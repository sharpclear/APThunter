# 订阅自动检测优化说明

## 优化背景

**问题**：数据文件延迟到达
- 1月8日的数据要到1月9日才能获取
- 不确定1月9日几点能获取到1月8日的数据
- 需要保证没有遗漏任何一天的检测数据
- 尽量保证新到的数据都能被检测

## 优化策略

### 1. 数据延迟配置
- 添加了 `DATA_DELAY_DAYS` 环境变量（默认1天）
- 表示数据文件延迟多少天到达系统

### 2. 增量检测机制
- **首次检测**：检测前x天的数据（排除今天及最近DATA_DELAY_DAYS天）
- **后续检测**：从上次检测的日期+1开始，到可用的最新日期结束
- **避免重复**：通过记录上次检测日期，确保不重复检测

### 3. 日期范围计算逻辑

#### Daily 订阅示例（1月8日创建，DATA_DELAY_DAYS=1）

**首次执行（1月9日）**：
- 结束日期：1月9日 - 1天 = 1月8日（排除今天，因为今天的数据还没到）
- 开始日期：1月8日（只检测1天）
- **检测数据**：1月8日的数据（此时1月8日的数据已经到达）

**第二次执行（1月10日）**：
- 上次检测日期：1月8日
- 结束日期：1月10日 - 1天 = 1月9日
- 开始日期：1月8日 + 1 = 1月9日
- **检测数据**：1月9日的数据

**第三次执行（1月11日）**：
- 上次检测日期：1月9日
- 结束日期：1月11日 - 1天 = 1月10日
- 开始日期：1月9日 + 1 = 1月10日
- **检测数据**：1月10日的数据

#### Weekly 订阅示例（1月8日创建，DATA_DELAY_DAYS=1）

**首次执行（1月15日）**：
- 结束日期：1月15日 - 1天 = 1月14日
- 开始日期：1月14日 - 6天 = 1月8日
- **检测数据**：1月8日到1月14日的数据（共7天）

**第二次执行（1月22日）**：
- 上次检测日期：1月14日
- 结束日期：1月22日 - 1天 = 1月21日
- 开始日期：1月14日 + 1 = 1月15日
- **检测数据**：1月15日到1月21日的数据（共7天）

## 关键改进

### 1. 增量检测
- 从Task记录中获取上次检测的日期范围
- 下次检测从上次检测的结束日期+1开始
- 确保不遗漏、不重复

### 2. 数据延迟处理
- 排除今天及最近DATA_DELAY_DAYS天的数据
- 确保检测的数据已经到达系统

### 3. 缺失数据处理
- 如果某天数据不存在，跳过该天，继续处理其他日期
- 不会因为没有数据而报错

### 4. 范围限制
- Daily：每次最多检测1天
- Weekly：每次最多检测7天
- Monthly：每次最多检测30天
- 避免一次性检测太多数据

## 数据路径

数据文件路径格式：
```
{DAILY_DATA_DIR}/{YYYY-MM}/{YYYY-MM-DD}-domain.zip
```

其中：
- `DAILY_DATA_DIR`：可通过环境变量配置，默认为 `backend/app/daily_data`
- 文件结构：zip文件包含 `dailyupdate.txt`，每行一个域名

## 配置说明

### 环境变量

1. **DATA_DELAY_DAYS**（默认：1）
   - 数据文件延迟天数
   - 例如：设置为1表示今天的数据明天才能获取

2. **DAILY_DATA_DIR**（可选）
   - 每日数据文件目录
   - 默认为：`backend/app/daily_data`

## 验证方法

1. **查看日志**：
   ```bash
   tail -f logs/app.log | grep -i subscription
   ```
   应该能看到：
   - `订阅 {id} 上次检测到: {date}`
   - `订阅 {id} 本次检测日期范围: {start} 到 {end}`

2. **检查数据库**：
   ```sql
   -- 查看任务记录
   SELECT task_id, created_at, extra->>'$.dateRange' as date_range
   FROM tasks 
   WHERE extra->>'$.subscription_id' = 'S你的订阅ID'
   ORDER BY created_at DESC;
   ```

3. **手动触发**：
   ```bash
   curl -X POST http://localhost:8000/api/subscriptions/trigger \
     -H "X-User-Id: 你的用户ID" \
     -H "Authorization: Bearer 你的token"
   ```

## 注意事项

1. **服务运行要求**：调度器只在服务运行时工作
2. **数据延迟配置**：根据实际数据到达时间调整 `DATA_DELAY_DAYS`
3. **时区问题**：代码使用 UTC 时间，注意时区转换
4. **缺失数据**：如果某天数据不存在，会跳过该天，不会报错
